· 2021-03-29 ·

MVCC

Multi version concurrency control

In RC and RR, the session can see the record before other session's updating because of MVCC and undo-log.


Good transaction practice

submit outside loop
do query that uses large lock range later in transaction
two transactions get resources in same order
choose RC is phantom read is fine
don't mix innodb and mysiam table (mysiam doesn't support transaction and rollback)

    phase 2 commit/rollback
